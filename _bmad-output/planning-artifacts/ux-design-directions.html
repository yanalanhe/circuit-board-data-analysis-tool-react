Comparison
Direction Evaluation
Summary of trade-offs between directions A (Dark Professional) and B (Light Clean). Direction C is not a standalone visual direction — it demonstrates error states within Direction A.

Direction A — Dark Professional Recommended

Dark neutral background, blue accent, color-coded execution states. Matches the VS Code / GitHub Actions aesthetic engineers use daily. Reduces eye strain for long lab sessions.


✓ Familiar to technical users
✓ Execution state colors are highly readable on dark backgrounds
✓ Streamlit dark theme — zero CSS required for baseline
✓ Chart output (matplotlib) looks polished on dark background

✗ May feel less approachable to non-technical observers (Morgan)
✗ Requires CSS override for step-status color injection

Best for: Long lab sessions, technical primary users

Direction B — Light Clean

Light neutral background, same blue accent. Clean, professional feel. Better for presentations and screen-sharing with non-technical stakeholders. Matches default Streamlit light mode.


✓ More approachable for non-technical observers
✓ Better for printing or screen-sharing in bright rooms
✓ Streamlit light theme — zero CSS required

✗ Status colors (green, amber, red) less punchy on light background
✗ matplotlib defaults produce dark-background charts by default — mismatch
✗ Less consistent with the technical tools (VS Code, Jupyter dark) engineers use alongside

Best for: Demos, presentations, bright environments
Recommendation

Direction A (Dark Professional) is recommended as the default. It aligns with the technical aesthetic engineers already work in, makes execution state colors clearly readable, and avoids the matplotlib dark-chart-on-light-background mismatch. The config.toml theme setting makes this a single-line configuration, not a custom build. Direction B can be offered as an optional light mode in a future iteration via Streamlit's built-in theme toggle.
